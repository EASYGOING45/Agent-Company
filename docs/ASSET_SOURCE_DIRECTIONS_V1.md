# Agent-Company — Asset Source Directions V1

## 1. 目标

这份文档用于明确：
- 哪些素材适合直接寻找公开可用资源
- 哪些素材适合参考后再转成项目自己的表达
- 哪些素材可以从旧项目中借鉴“构图逻辑”而不是直接搬运

---

## 2. 素材来源策略

## 2.1 可直接寻找公开可用素材的部分
这些内容更适合找公开可用像素素材：
- 办公室背景
- 桌子 / 椅子 / 工位 / 屏幕
- 植物 / 海报 / 装饰
- 像素按钮 / UI 小框 / badge
- 基础状态图标

原因：
- 通用性强
- 与角色 IP 绑定较弱
- 更容易找到明确许可的像素素材

## 2.2 适合“参考后转化”的部分
这些内容更适合参考气质与构图，再转成自己的表达：
- 主空间舞台氛围
- 主人区域设计
- 成员展示方式
- 总结层视觉组合
- co-working 式空间感

原因：
- 这些部分决定产品辨识度
- 直接照搬会让产品不够独立

## 2.3 当前案例主角素材（菲比）
这部分要更谨慎。

当前建议：
- 先找角色参考图，用于校准气质
- 首页原型阶段可以先采用“案例参考 + 占位表现”
- 后续逐步转成更统一的项目资产表达

也就是说：
- 当前阶段可参考
- 中后期要项目化

---

## 3. 旧项目的正确使用方式

StarOffice 当前最适合作为：
- 空间层次参考
- 像素办公室舞台参考
- 家具区域划分参考
- 角色在场景中的存在方式参考

不适合作为：
- 直接产品素材来源
- 直接命名体系来源
- 直接角色系统来源

---

## 4. 建议优先寻找的素材类型

### 优先级 A
- 像素办公室背景
- 像素家具套件
- 像素 UI 元件

### 优先级 B
- 主人案例参考图（菲比）
- 默认成员占位
- 默认访客占位

### 优先级 C
- 动画素材
- 更完整的空间装饰
- 多主题替换素材

---

## 5. 当前结论

素材搜集应该遵循：

> 通用空间与 UI 素材优先找公开可用资源；角色气质与主舞台则以参考转化为主；旧项目只借结构，不借身份。
