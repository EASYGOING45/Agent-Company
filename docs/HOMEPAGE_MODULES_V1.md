# Agent-Company — Homepage Modules V1

## 1. 目标

这份文档用于把首页从“概念”推进到“可直接做静态原型”的程度。

目标：
- 明确首页每个模块存在的理由
- 明确模块优先级
- 明确哪些模块是 V1 必做，哪些可后置

---

## 2. 首页模块总览

首页模块分为 5 个层级：

1. Top Bar（顶部信息条）
2. Main Stage（主空间舞台）
3. Member Rail（成员带）
4. Activity Feed（状态流）
5. Summary Dock（总结层）

---

## 3. 模块定义

## 3.1 Top Bar
### 作用
建立“这是一家 Agent 公司”的系统感。

### 建议内容
- 公司名：Agent-Company
- 当前主人：当前案例为菲比
- 当前时间
- 在线人数
- 今日概览（如：5 位在线 / 2 位工作中 / 1 份总结）
- 快速入口：成员 / 总结 / 设置

### V1 优先级
**必做**

---

## 3.2 Main Stage
### 作用
首页灵魂模块，承担产品记忆点。

### 建议内容
- 主人角色（当前案例为菲比）
- 像素办公室主场景
- 在线 Agent 的空间分布
- 当前主要状态氛围（工作 / 待命 / 研究 / 汇报中）

### 设计原则
- 不堆太多控件
- 保持舞台感
- 以“被观看的公司空间”为主

### V1 优先级
**最优先，必须打磨**

---

## 3.3 Member Rail
### 作用
让“公司里有哪些成员”随时可见。

### 建议内容
- 成员迷你卡
- 头像 / 小立绘
- 名称
- 身份标签
- 在线状态点

### 设计原则
- 轻量
- 不抢主空间
- 支持点击展开详情

### V1 优先级
**必做**

---

## 3.4 Activity Feed
### 作用
让首页呈现“公司在流动”。

### 建议内容
- 某成员刚上线
- 某成员刚签到
- 某成员切换状态
- 某成员提交总结
- 某成员异常恢复

### 设计原则
- 更像动态流，不像审计日志
- 信息短句化
- 时间感清楚

### V1 优先级
**必做**

---

## 3.5 Summary Dock
### 作用
表达“这家公司真的有产出”。

### 建议内容
- 今日总结摘要
- 最新汇报
- 最近产出卡片
- 查看完整日报入口

### 设计原则
- 比状态流更稳重
- 有内容块感
- 不要过于聊天化

### V1 优先级
**必做，但可先做简版**

---

## 4. V1 / V1.5 / V2 划分

## V1 必做
- Top Bar
- Main Stage
- Member Rail
- Activity Feed
- Summary Dock（简版）

## V1.5 可做
- 成员详情展开层
- 主人专属区域说明
- 今日概览的小统计强化

## V2 再做
- 完整成员页
- 完整总结页
- 配置页
- 多公司主题切换
- 多主人模板

---

## 5. 当前结论

首页原型阶段应当遵循：

> 先把主空间、成员、动态、总结四件事做出呼吸感，再考虑复杂的后台功能。
