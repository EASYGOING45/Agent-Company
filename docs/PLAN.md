# Agent-Company — PLAN

## 项目定位

Agent-Company 是一个面向 **Agents / OpenClaw** 的数字工作室项目。

它不是传统后台，也不是普通状态面板，而是一个带有角色感、空间感和运营感的 **Agent 公司 / Agent 工作室界面**。

核心设定：
- 公司主人 / 默认主角是 **菲比（Phoebe）**
- 不同 Agent 以成员、访客、同事、搭档等身份加入公司
- 页面需要同时承载“状态展示、签到记录、总结管理、角色配置”四类能力
- 产品既能日常自用，也能作为展示型作品存在

一句话描述：

> 一个以菲比为主角的 Agent 公司界面，让多个 Agents / OpenClaw 像在同一家公司里工作一样被看见、被管理、被展示。

## 功能清单

### P0 基础骨架
- 新项目初始化，不继承旧项目命名体系
- 建立本地 `PLAN.md` / `INTRO.md`
- 建立飞书项目文档
- 确立菲比为默认主角
- 确立 Agent-Company 的世界观、产品文案与视觉方向

### P1 角色系统
- 主人 / 成员 / 访客角色模型
- 角色基础资料：名称、身份、头像、主视觉、简介、颜色
- 主角色（菲比）默认配置
- 角色头像 / 立绘 / 动画资源可配置
- 角色显示逻辑统一，不再随机分配头像

### P2 工作状态系统
- Agent 签到 / 上线
- 当前状态记录（待命、工作、研究、开会、故障等）
- 状态更新时间记录
- 最近动态展示
- 在线 / 离线 / 异常识别

### P3 管理与总结系统
- 每日总结 / 工作汇报展示
- Agent 备注与简介管理
- 公司视角的成员面板
- 简单运营看板（谁在线、谁在做事、今天做了什么）

### P4 展示与体验系统
- 像素风或轻游戏化办公空间
- 手机端 / 桌面端兼容
- 适合直播、演示、展示
- 默认视觉主角为菲比
- 支持后续扩展更多办公室区域 / 房间

#### 当前设计方向（第一版）
参考 co-working 的产品感觉，Agent-Company 应该更像：
- 一个“正在运转的 Agent 公司空间”
- 强调成员共处、工作流动、在线感、存在感
- 不要做成冰冷的管理后台，而要做成“有人味的 Agent 办公空间”

第一版视觉与产品方向建议：
- **关键词**：co-working、company、presence、soft management、character-driven
- **气质**：温暖、轻游戏化、可展示、带角色感
- **界面原则**：空间 > 卡片；角色 > 表格；状态流动 > 静态列表
- **核心记忆点**：菲比作为公司主人，在一个可被观看的 Agent 办公空间里管理大家

第一版信息架构建议：
1. **公司主空间（首页）**
   - 展示菲比与主要在线 Agent
   - 看到谁在上班、谁在摸鱼、谁在工作、谁刚刚汇报
2. **成员面板**
   - 成员列表 / 角色卡 / 身份标签 / 在线状态
3. **状态流面板**
   - 最近签到、状态更新、异常、离线、上线
4. **总结面板**
   - 今日总结 / 工作汇报 / 最近产出
5. **角色配置层**
   - 头像、立绘、颜色、称号、身份配置

第一版页面结构建议：
- 顶部：公司名称、时间、在线人数、今日概览
- 中央：主空间 / 办公区域 / 菲比主视觉
- 右侧：最近动态 / 状态流
- 左侧或底部：成员卡片 / 总结入口 / 快速操作
