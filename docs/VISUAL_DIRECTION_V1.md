# Agent-Company — Visual Direction V1

## 1. 目标感觉

Agent-Company 的页面第一眼应该给人的感觉不是：
- 一个控制台
- 一个仪表盘
- 一个 AI 工具后台

而应该是：
- 一个正在运转的数字公司
- 一个有主人、有成员、有氛围的 Agent 办公空间
- 一个既适合自己看，也适合展示给别人看的页面

---

## 2. 视觉关键词

- Pixel office
- Co-working
- Presence
- Warm operations
- Character-first
- Soft game UI
- Live company feeling

---

## 3. 风格判断

这是一个 **像素风 + 展示型 + 角色驱动** 的产品。

所以风格应当避免两种极端：

### 不要变成纯后台
不要大量使用：
- 数据表
- 统计卡片海
- 大量数字 KPI
- 传统企业后台导航

### 也不要变成纯游戏 HUD
不要变成：
- 纯战斗 UI
- 花哨技能面板
- 过度二游化覆盖层
- 一堆喧闹装饰压住信息层级

正确方向是两者之间：

> 一个有像素风空间感、但同时有产品信息组织能力的 Agent 公司界面。

---

## 4. 主视觉原则

## 4.1 主人必须有“主场感”
当前案例里，菲比必须是首页主视觉中心。

但这种中心不是巨幅立绘糊一层，而是：
- 在公司空间里拥有明确主位
- 有自己的办公区域 / 管理区域
- 和页面结构形成自然融合

## 4.2 空间先于装饰
主视觉不该只是插画区。

正确方式是：
- 先有空间
- 再有角色
- 再有动态信息流

让页面看起来像：
- 菲比真的在这个公司里
- 其他成员也真的在这里活动

---

## 5. 组件气质建议

## 5.1 成员卡片
成员卡片应该：
- 小而精
- 不抢首页舞台
- 重点展示身份、状态、头像

适合做成：
- 像素 badge + 小卡片
- 带身份标签和状态点
- 可展开查看更多信息

## 5.2 状态流
状态流应该：
- 更像“公司动态”
- 少一点日志味
- 多一点生活感与现场感

例如：
- 菲比刚开始安排工作
- 某成员刚提交总结
- 某成员刚上线
- 某成员进入待命

## 5.3 总结模块
总结模块应该：
- 比成员卡更稳重
- 有内容感
- 不是聊天气泡，而是工作产出卡片

---

## 6. 色彩方向

建议围绕菲比案例先做一版默认主题。

### 主色
- 柔金 / 浅金
- 代表主人、焦点、温暖感

### 辅色
- 浅蓝 / 天空蓝
- 代表辅助信息、空间空气感、在线感

### 结构色
- 墨蓝 / 深灰蓝 / 暖灰
- 用于背景、边框、层级骨架

### 状态色
- 在线：偏绿色或清亮蓝绿
- 工作中：暖金 / 活跃黄
- 待命：低饱和蓝灰
- 异常：偏橙红，但不要刺眼

---

## 7. 版式方向

## 7.1 大舞台 + 小模块
V1 首页应当采用：
- 中央大舞台
- 周围小模块支持

而不是平均切卡。

## 7.2 重点永远在中央主空间
任何成员卡、状态流、总结卡，都不应抢走中央空间的主导感。

首页记忆点应该始终是：
- 菲比
- 办公空间
- 公司正在运转

---

## 8. 素材方向

可参考的素材来源方向：
- 像素办公室背景素材
- 像素家具、植物、桌椅、屏幕、工位
- 像素人物 idle / work 动画
- 可公开可用的像素 UI 元件

旧 StarOffice 项目可参考：
- 空间构图
- 像素办公氛围
- 场景层次关系

但新项目需要：
- 新命名
- 新角色系统
- 新主人模型
- 新视觉组织方式

---

## 9. 当前结论

Agent-Company V1 的最佳视觉方向是：

> 用 StarOffice 那类像素办公室表达作为参考骨架，再用“配置化主人 + 角色系统 + co-working 公司气质”把它提升成一个真正的新产品。
