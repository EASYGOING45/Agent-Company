# Agent-Company — Material Research Log V1

## 当前进展

这一轮的目标不是一次性找齐全部素材，而是先把“首页静态原型”需要的素材搜集策略固定下来。

目前已经完成：
- 明确首页五大模块
- 明确首页第一批素材 shortlist
- 明确旧 StarOffice 中哪些内容适合作为参考骨架
- 明确新项目需要独立的主人模型、角色素材命名和视觉组织方式

---

## 当前素材搜集策略

### 1. 空间类素材
优先找：
- 像素办公室背景
- 办公桌 / 工位 / 屏幕 / 植物 / 沙发等元素
- 可拼装的办公室小物件

用途：
- 支撑首页 Main Stage
- 做出空间层次

### 2. UI 类素材
优先找：
- 像素按钮
- badge / 标签框
- 小卡片底板
- 状态点 / 图标

用途：
- 成员带
- 状态流
- 总结层

### 3. 角色类素材
优先处理：
- 当前案例主人：菲比
- 默认成员占位
- 默认访客占位

注意：
- 产品必须配置化
- 当前只把菲比作为默认案例，不把她写死成唯一主角

---

## 旧项目可迁移的“参考价值”

来自 StarOffice，可直接借鉴：
- 像素办公室空间氛围
- 家具布局层次
- 主舞台构图逻辑
- 人物处于空间中的展示方式

不直接继承：
- Star 命名
- 旧主人逻辑
- 随机访客头像体系
- 旧产品世界观
