# Agent-Company 鸣潮化改造计划

**参考项目**：Miniverse（https://github.com/ianscott313/miniverse）
**目标**：基于 Miniverse 架构，改造成鸣潮世界观的 Agent 元宇宙

---

## Miniverse 核心架构（值得借鉴）

```
Agent（任意） → POST /api/heartbeat → Local Server → WebSocket → Canvas 前端
```

### 技术栈
- **前端**：HTML5 Canvas + Vite，纯前端无框架
- **后端**：Node.js HTTP + WebSocket 服务器
- **Agent 接入**：只需 POST heartbeat，无需注册
- **渲染**：2D Canvas，image-rendering: pixelated

### 核心功能
1. **心跳机制**：Agent 定期 POST 状态，30秒无心跳自动 offline
2. **实时推送**：WebSocket 推送所有 Agent 状态变化
3. **Canvas 渲染**：像素风角色、动画、寻路
4. **可交互物品**：白板、咖啡机、监控等
5. **语音气泡**：显示当前任务

## 改造实施计划

### Phase 1：前端 Canvas 化（借鉴 Miniverse）

**目标**：把现在的 DOM 基础版本，改成 Canvas 渲染

**技术方案**：
- 使用 HTML5 Canvas 替代部分 DOM 渲染
- 借鉴 Miniverse 的 sprite 系统
- 保持像素风 + CRT 效果

**文件结构（改造后）**：
```
src/
  canvas/
    engine.js        # Canvas 渲染引擎
    sprites.js       # Sprite 动画系统
    pathfinding.js   # A* 寻路（可选）
    world.js         # 世界/场景管理
  components/
    AgentRenderer.js # Agent 渲染
    RoomRenderer.js # 房间背景渲染
  api/
    heartbeat.js     # 心跳上报
    websocket.js     # WebSocket 客户端
  app.js             # 主入口
public/
  assets/
    sprites/         # 像素角色图
    tiles/           # 地图瓦片
```

**改动点**：
1. 新增 Canvas 渲染层
2. Agent 用 Canvas 绘制（带动画）
3. 背景/装饰用 Canvas 渲染
4. 保留 UI overlay（房间 Tab、Toast 等）

### Phase 2：后端服务（借鉴 Miniverse Server）

**目标**：搭建 Node.js HTTP + WebSocket 服务器

**技术方案**：
- 使用 Node.js 原生 http + ws（不用额外框架）
- 或者直接借鉴 Miniverse 的 server 包

**API 设计**：
```javascript
// Agent 心跳上报（核心！）
POST /api/heartbeat
{
  agent: "phoebe",
  state: "working",      // working/idle/thinking/error/offline
  task: "元宇宙 2.0 开发",
  room: "lobby"
}

// 获取所有 Agent 状态
GET /api/agents

// WebSocket 实时推送
ws://localhost:3000/ws
```

**数据存储**：
- 内存存储（简单够用）
- 30秒无心跳自动 offline

### Phase 3：鸣潮素材包

**目标**：创建鸣潮风格的角色 sprite 和场景

**资源需求**：
1. **角色 Sprites**：
   - 菲比（站立、走路、工作中、待机）
   - 其他 Agent 通用角色
   
2. **场景 Tiles**：
   - 修会地板、墙壁
   - 共鸣仪工作台
   - 棱镜装饰
   - 教堂窗户

3. **物品**：
   - 通讯棱镜（替代 intercom）
   - 能量站
   - 任务板

### Phase 4：Agent 接入文档

**目标**：让其他 Agent（OpenClaw/菲比）能接入

**接入方式**（借鉴 Miniverse）：
```bash
# 简单 curl 方式
curl -X POST http://localhost:3000/api/heartbeat \
  -H "Content-Type: application/json" \
  -d '{"agent":"phoebe","state":"working","task":"任务中","room":"lobby"}'
```
