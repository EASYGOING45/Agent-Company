# Agent-Company — Hero Reference Composition V1

## 1. 目标

定义首页第一屏（Hero / Main Stage）在参考搜集与原型阶段，应该由哪些视觉元素组成。

重点不是精细视觉稿，而是先确定：
- 主舞台必须出现什么
- 什么能后补
- 第一屏怎么建立产品气质

---

## 2. 第一屏必须出现的元素

## 2.1 公司身份
必须让用户第一眼知道：
- 这是 Agent-Company
- 当前案例主人是菲比
- 这不是普通后台，而是一个 Agent 公司空间

可承载位置：
- 顶部信息条
- 主空间标题区

## 2.2 主人主位
当前案例里，菲比必须有明确主位。

主位不一定是超大立绘，但必须：
- 明确是视觉中心
- 和办公室空间融合
- 能体现“公司主人”而非普通成员

## 2.3 办公空间背景
第一屏必须有可被感知的空间，而不是一层抽象色块。

至少要具备：
- 办公区
- 可放置成员的位置
- 有深浅层次的空间氛围

## 2.4 在线成员存在感
第一屏不需要展示所有成员，但必须出现少量在线成员存在感。

建议：
- 2~4 个活跃 Agent
- 用小体量角色点位或成员卡提示
- 重点强调“公司正在运转”

## 2.5 动态感
即使是静态原型，也要让人感觉页面是活的。

可通过以下方式表达：
- 状态流短句
- 在线状态点
- 时间信息
- 总结摘要卡

---

## 3. 第一屏建议组合

建议把首页第一屏理解成 4 层组合：

### 层 1：背景空间
办公室背景 + 家具 + 分区层次

### 层 2：主人层
菲比主位 / 主人区域 / 主视觉重心

### 层 3：成员层
2~4 个活跃 Agent 的存在提示

### 层 4：信息层
顶部条 / 状态流 / 总结摘要

---

## 4. 第一屏不能犯的问题

### 不要只有 UI，没有空间
否则会变成普通后台。

### 不要只有角色，没有产品信息
否则会变成角色展示页。

### 不要堆太多成员
否则会破坏舞台重心。

### 不要一开始就做太复杂
第一屏最重要的是把“公司主人 + 空间 + 运行感”先做出来。

---

## 5. 当前结论

首页第一屏最重要的不是信息多，而是：

> 让用户一眼看到“菲比在一个 Agent 公司空间里管理大家，而这个空间正在运转”。
