# Agent-Company — Reference Source Types V1

## 1. 目标

明确接下来找参考与素材时，应该优先看哪几类来源。

这里先定义“类型”，不强行列一堆具体链接，避免早期搜集失控。

---

## 2. 推荐优先看的来源类型

## 2.1 像素办公室 / 室内空间素材包
适合找：
- 办公室背景
- 桌椅 / 工位 / 屏幕 / 家具
- 室内装饰元素

用途：
- 支撑首页主空间
- 提供空间气氛基底

## 2.2 像素 UI / RPG UI / 模拟经营 UI 素材
适合找：
- 按钮
- badge
- 信息框
- 小卡片底板
- 状态图标

用途：
- 成员带
- 状态流
- 总结层

## 2.3 像素人物 idle / work 参考
适合找：
- 人物站立状态
- 工作状态
- 轻量循环动画
- 小尺寸角色在场景中的展示方式

用途：
- 成员在空间中的“存在感”参考
- 后续角色表现形式参考

## 2.4 共创空间 / co-working 网站参考
适合看：
- 信息结构
- 模块关系
- 公司感 / 社区感 / 运营感表达

用途：
- 决定产品怎么“像一家公司”
- 避免页面做成单纯素材拼贴

## 2.5 当前案例角色参考（菲比）
适合看：
- 气质
- 配色
- 姿态
- 主人感与主角感

用途：
- 当前案例主题校准
- 默认视觉方案参考

---

## 3. 搜集时的规则

### 规则 1
先找“能撑起首页第一屏”的素材，不要一开始搜太散。

### 规则 2
优先找通用空间和 UI，再处理角色细节。

### 规则 3
角色参考可以先粗一点，但主舞台背景一定要尽快定方向。

### 规则 4
旧 StarOffice 继续作为结构参考，但不作为产品身份来源。

---

## 4. 下一步

下一轮实际搜集时，优先按这几类去找：
1. 像素办公室背景 / 家具
2. 像素 UI 小组件
3. 菲比案例参考
4. 小尺寸角色表现参考

---

## 5. 结论

参考搜集不应该是“无穷无尽找图”，而应该是：

> 为首页第一屏服务的定向搜集。
