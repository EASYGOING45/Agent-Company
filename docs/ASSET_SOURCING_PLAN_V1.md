# Agent-Company — Asset Sourcing Plan V1

## 1. 素材策略

Agent-Company 的素材策略分三层：

1. **可直接参考旧 StarOffice 的空间结构**
2. **寻找新的可公开可用像素素材**（办公室、家具、植物、屏幕、UI）
3. **主角色素材单独处理**（当前案例为菲比）

---

## 2. 旧项目可参考项

可参考但不直接继承旧世界观：
- 办公室空间构图
- 桌椅、沙发、植物、服务器区域等布局思路
- 像素办公室氛围
- 成员在空间中展示的方式

旧项目前端中已观察到的参考素材类型：
- office_bg / office_bg_small
- desk
- sofa
- plants / flowers / posters
- coffee_machine
- serverroom
- guest_anim_* / guest_role_*
- error_bug / sync_animation

这些内容适合作为：
- 构图参考
- 功能分层参考
- 可迁移思路参考

不适合直接照搬为最终产品素材。

---

## 3. Agent-Company 当前需要的素材分类

### 3.1 空间素材
- 办公室背景
- 桌椅 / 工位
- 植物 / 装饰
- 屏幕 / 墙面 / 标识
- 主人区域 / 管理区

### 3.2 UI 素材
- 像素按钮
- 标签框
- 状态条
- 小型信息面板
- 成员卡背景

### 3.3 角色素材
- 主角色头像
- 主角色主视觉
- 主角色 idle / work 状态资源
- 成员默认占位头像
- 访客默认占位头像

### 3.4 动态氛围素材
- 打字 / 工作动画
- 在线状态闪烁
- 异常提醒图标
- 小型环境动效（屏幕、植物、灯光）

---

## 4. 当前原则

### 4.1 角色素材配置化
- 主人是谁，不绑定到素材命名本身
- 不要再出现 `star_idle` 这种产品层写死命名
- 建议未来命名方式：
  - `owner_idle`
  - `owner_working`
  - `agent_default_idle`
  - `guest_default_avatar`
  - 或按角色目录组织

### 4.2 素材版权与来源要清楚
- 优先：公开可用 / 明确许可素材
- 次选：可作为参考图的公开图源，用于二次设计而非直接商用搬运
- 对主角色（菲比）尤其要谨慎，建议用于参考、校准风格，并逐步转成项目自己的表达

---

## 5. 下一步执行建议

1. 先整理一份“可直接复用的空间参考清单”
2. 再找一批公开可用的像素办公室素材
3. 最后单独处理主角色（菲比案例）的视觉方案

目标不是先把所有素材找齐，而是先把：
- 首页主空间所需素材
- 成员卡所需素材
- 状态流基础 UI 素材

这三部分先凑齐，足够支撑第一版静态原型。
